## Reglas técnicas

### Arquitectura
- Presentation solo consume componentes y navegación.
- No requiere UseCases adicionales (solo navegación).
- Usar `next/navigation` para navegación en App Router.
- Usar `NextLink` de Next.js para enlaces internos.

### Navegación
- Enlaces de adopción: `router.push('/adopt?species=dog')` o `NextLink`
- Enlace de tienda: `router.push('/shop')` o `NextLink`
- Enlace de registro: `router.push('/register/foundation')` o `NextLink`
- Email: `mailto:contacto@amigogigante.org`

### Query Params
- `species=dog` → filtrar por especie perro
- `species=cat` → filtrar por especie gato
- `urgentOnly=true` → filtrar casos urgentes
- El catálogo de adopción debe leer estos params y aplicar filtros.

### Traducciones
- Usar `useTranslations('common')` para textos del footer.
- Keys necesarias:
  - `footer.adoption`
  - `footer.community`
  - `footer.contact`
  - `footer.contactEmail`
  - `footer.becomeFoundation`
  - `footer.links.adoption.*`
  - `footer.links.community.store`

### Iconos
- `EmailRoundedIcon` de MUI para email
- `BusinessRoundedIcon` de MUI para fundación
- Tamaño: `fontSize: 18`
- Color: `text.secondary`

### Dependencias / Paquetes
- No agregar dependencias nuevas.
- Usar componentes existentes de MUI.
